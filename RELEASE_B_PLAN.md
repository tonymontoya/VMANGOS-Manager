## Research Findings

### 1. Database Backup Reality Check ✅

**Test Results from Live System:**
```
Database Sizes:
- world:    150.55 MB (169 tables, largest: item_template 35MB)
- logs:       0.34 MB (15 tables)
- characters: 0.24 MB (55 tables)
- auth:       0.18 MB (11 tables)

Backup Performance:
- Full world dump:    117MB raw → 28MB compressed (4.25x ratio) in 1.3s
- Full auth dump:      13KB raw → 2.4KB compressed (5.5x ratio) in 0.03s
- Restore test:        Successful (auth → test db → verified tables)

Key Finding: MySQL binlog is OFF - incremental backups not possible without configuration change
```

**Impact on Plan:**
- **Defer incremental backups to Release C** (requires enabling binlog)
- **Full backups only for Release B** - fast enough given compression ratios
- **Estimated backup time:** ~2s for all databases on this hardware

### Phase 2: Backup System (Issue #8) - 2-3 days

#### 2.1 Database Schema

```sql
-- Create vmangos_mgr schema if not exists
CREATE DATABASE IF NOT EXISTS vmangos_mgr;

-- Backup metadata table
CREATE TABLE IF NOT EXISTS vmangos_mgr.backups (
    id INT AUTO_INCREMENT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    type ENUM('full') DEFAULT 'full',  -- incremental deferred to Release C
    databases JSON NOT NULL,
    size_bytes_raw BIGINT NOT NULL,
    size_bytes_compressed BIGINT NOT NULL,
    compression_ratio DECIMAL(4,2) NOT NULL,
    checksum VARCHAR(64) NOT NULL,
    path VARCHAR(512) NOT NULL,
    status ENUM('active', 'restoring', 'restored', 'expired', 'failed') DEFAULT 'active',
    expires_at TIMESTAMP NOT NULL,
    duration_seconds DECIMAL(6,2),
    INDEX idx_status (status),
    INDEX idx_expires (expires_at)
);

-- Restore history table
CREATE TABLE IF NOT EXISTS vmangos_mgr.restore_history (
    id INT AUTO_INCREMENT PRIMARY KEY,
    backup_id INT NOT NULL,
    restored_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    target_database VARCHAR(64) NOT NULL,
    status ENUM('success', 'failed') NOT NULL,
    error_message TEXT,
    FOREIGN KEY (backup_id) REFERENCES backups(id)
);
```

#### 2.2 Library: lib/backup.sh

**Functions:**
```bash
# Create backup
backup_create() -> Returns: backup_id, error code
  - Acquire global backup lock
  - Create backup directory /opt/mangos/backups/YYYYMMDD/
  - Dump each database with mysqldump --single-transaction
  - Compress with gzip (configurable level)
  - Calculate SHA256 checksum
  - Insert metadata to vmangos_mgr.backups
  - Release lock

# List backups
backup_list([--format json]) -> Returns: formatted list
  - Query vmangos_mgr.backups
  - Calculate age, size
  - Format as text or JSON

# Restore backup
backup_restore(backup_id, [--database name]) -> Returns: success/failure
  - Verify backup exists and checksum
  - Create restore lock
  - Stop relevant services (or use --no-stop flag)
  - Decompress and restore
  - Update restore_history
  - Release lock

# Prune old backups
backup_prune([--keep-days N]) -> Returns: count removed
  - Find expired backups
  - Delete files
  - Update status to 'expired'
```

#### 2.3 CLI Integration

```bash
vmangos-manager backup create [--compress-level 6] [--no-lock]
vmangos-manager backup list [--format json] [--all|--active]
vmangos-manager backup restore <id> [--database <name>] [--verify-only]
vmangos-manager backup prune [--keep-days 7] [--dry-run]
vmangos-manager backup verify <id>  # Checksum validation
```

#### 2.4 Automated Backup Timer

**File:** `/etc/systemd/system/vmangos-backup.timer`
```ini
[Unit]
Description=VMANGOS Daily Backup Timer

[Timer]
OnCalendar=daily
OnCalendar=*-*-* 03:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

**File:** `/etc/systemd/system/vmangos-backup.service`
```ini
[Unit]
Description=VMANGOS Database Backup
After=mysql.service

[Service]
Type=oneshot
ExecStart=/opt/mangos/manager/bin/vmangos-manager backup create
User=root
StandardOutput=journal
StandardError=journal
```

### Phase 3: Health Monitoring (Issue #10) - 1-2 days

#### 3.1 Database Schema

```sql
-- Health check results
CREATE TABLE IF NOT EXISTS vmangos_mgr.health_checks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    checked_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    auth_active BOOLEAN NOT NULL,
    world_active BOOLEAN NOT NULL,
    auth_restarts_1h INT DEFAULT 0,
    world_restarts_1h INT DEFAULT 0,
    online_players INT DEFAULT 0,
    disk_usage_percent INT NOT NULL,
    disk_free_gb DECIMAL(6,2) NOT NULL,
    db_reachable BOOLEAN NOT NULL,
    db_response_ms INT,
    errors JSON,
    INDEX idx_checked_at (checked_at)
);

-- Alert history
CREATE TABLE IF NOT EXISTS vmangos_mgr.alerts (
    id INT AUTO_INCREMENT PRIMARY KEY,
    triggered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    alert_type VARCHAR(64) NOT NULL,
    severity ENUM('info', 'warning', 'critical') NOT NULL,
    message TEXT NOT NULL,
    resolved_at TIMESTAMP NULL,
    INDEX idx_unresolved (resolved_at, severity)
);
```

#### 3.2 Library: lib/health.sh

**Functions:**
```bash
# Check service health
health_check_services() -> { auth_active, world_active, auth_restarts, world_restarts }
  - systemctl is-active
  - journalctl --since "1 hour ago" restart count

# Check player metrics
health_check_players() -> { online_count, total_accounts }
  - Query auth.account.online
  - Query auth.account count

# Check system resources
health_check_system() -> { disk_percent, disk_free_gb, db_reachable, db_response_ms }
  - df /opt/mangos
  - mysqladmin ping (measure response time)

# Run full health check
health_check([--alert]) -> Writes to health_checks table, optionally triggers alerts
  - Combine all checks
  - Insert to database
  - Evaluate alert conditions
  - Trigger alerts if --alert specified

# Check alerts
health_alerts_list([--unresolved]) -> List of alerts
```

**Alert Conditions:**
| Condition | Severity | Check |
|-----------|----------|-------|
| world_active = false | CRITICAL | Every check |
| auth_active = false | CRITICAL | Every check |
| world_restarts_1h > 10 | WARNING | Every check |
| disk_usage_percent > 90 | WARNING | Every check |
| disk_usage_percent > 95 | CRITICAL | Every check |
| db_reachable = false | CRITICAL | Every check |

#### 3.3 CLI Integration

```bash
vmangos-manager health check [--alert] [--format json]
vmangos-manager health status [--format json] [--last N]
vmangos-manager health alerts [--unresolved] [--resolve <id>]
```

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Backup during world update | Global lock prevents concurrent operations |
| Disk full during backup | Pre-backup disk space check |
| Restore corruption | Checksum verification before restore |
| Service restart loop | Health check tracks restart count |
| Backup file permission issues | Enforce 600 mode on backup files |
| Database connection failure | Retry logic with exponential backoff |
